# Development

## Prerequisites

Install [asdf](https://asdf-vm.com/) and [just](https://github.com/casey/just), then:

```bash
just install-tooling    # installs all tools from .tool-versions via asdf
pre-commit install      # install git commit hooks
```

All tool versions (Terraform, Helm, kubectl, kind, Go, etc.) are pinned in `.tool-versions`.

## Cloud Provider Access

### AWS

Access is via Okta SSO. Configure once:

```bash
aws configure sso --profile infraex
# SSO start URL: https://d-9967231e9d.awsapps.com/start
# SSO region: eu-central-1
# Account ID: 4448-0410-6854
```

Login (short-lived credentials, re-run when expired):

```bash
aws sso login --profile infraex
aws sts get-caller-identity --profile infraex   # verify
```

> **Warning:** Regions **eu-west-2** and **eu-west-3** are reserved for CI and are nuked nightly. Never use them for local development.

### Azure

Access is via Okta with a `firstname.lastname@camundaittest.onmicrosoft.com` account.

```bash
az login                                          # browser-based MFA
az account set --subscription "<subscription>"   # set target subscription
```

For AKS access (kubectl):

```bash
az aks get-credentials --resource-group <RG> --name <CLUSTER>
kubelogin convert-kubeconfig -l azurecli         # required for AAD-integrated clusters
kubectl get nodes                                 # verify
```

kubelogin in `azurecli` mode relies on the Azure CLI token (valid ~24h). Re-run `az login` if kubectl auth fails the next day.

## Common Commands

```bash
# List all available recipes
just --list

# Run all AWS module unit tests
just aws-tf-modules-tests

# Run a single module test
just aws-tf-modules-test <testname>

# Run a test with verbose output
just aws-tf-modules-test-verbose <testname>

# Regenerate golden Terraform plan for a specific module
just regenerate-golden-file <module> <region> [additional args]

# Regenerate all golden files
just regenerate-golden-file-all
```

## Pre-commit Hooks

Hooks run automatically on `git commit`. To run manually:

```bash
pre-commit run --all-files
```

Hooks enforce:
- Terraform formatting (`terraform fmt`)
- Terraform docs generation (`terraform-docs` — auto-updates `README.md` in each module)
- tflint (with `.lint/tflint/.tflint.hcl` rules)
- yamllint + yamlfmt
- shellcheck
- actionlint
- Conventional commit format (scope optional)
- Go `mod tidy` + `gofmt`
- Trivy security scan
- Renovate config validation

## Terraform Conventions

- Root modules live in `{deployment}/terraform/` with subdirectories per concern (e.g. `cluster/`, `vpn/`)
- Reusable modules live in `{cloud}/modules/{name}/`
- All modules must have a `README.md` (auto-generated by `terraform-docs` hook)
- Test fixtures in `test/` — includes `ci/` (minimal config) and `golden/` (full plan output)

## Golden File Tests

Golden files (`tfplan-golden.json`) are Terraform plan outputs stored as JSON, with sensitive values redacted (ARNs, IPs, account IDs, versions). They are used for regression detection.

- Generate: `just regenerate-golden-file <module> <region>`
- The generator redacts sensitive fields automatically — verify redaction before committing.
- Never edit golden files by hand.

## Idempotent Kubernetes Resources

Always use dry-run + apply for `kubectl create` commands in scripts and CI:

```bash
kubectl create secret generic my-secret \
  --from-literal=password="$PASSWORD" \
  --namespace camunda \
  --dry-run=client -o yaml | kubectl apply -f -
```

This avoids failures when resources already exist, without silently swallowing real errors.

## Commit Messages

Conventional Commits are enforced by pre-commit hook:

```
<type>[optional scope]: <description>
```

Valid types: `feat`, `fix`, `refactor`, `test`, `docs`, `style`, `build`, `ci`, `chore`, `perf`, `deps`

Subject ≤ 120 chars, present tense, no period.

## Security Notes

Certain Trivy findings are deliberately ignored in `.lint/trivy/.trivyignore` (e.g. public cluster access, unencrypted storage) because these are expected in demo environments. Check this file before adding new ignores and document the reason.
